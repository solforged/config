# Conversation Note Style Guide

- Render through `notesctl`, not by hand.
- Keep frontmatter keys in this order: `title`, `captured_at`, `people`, `tags`, `conversation_id`, `source_url`, `style_guide`.
- Keep top-level headings in this order: `Summary`, `Decisions`, `Follow-Ups`, `Tasks`.
- Use concise prose in `Summary`.
- Keep `Decisions`, `Follow-Ups`, and `Tasks` as flat bullet lists.
- Prefer stable tags over conversational phrasing.
- Preserve source URLs and conversation IDs when they exist.
